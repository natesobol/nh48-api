# Peak Page Visual Review Checklist

Use this checklist after peak template or styling changes.

## Routes
- `/peak/mount-washington`
- `/peak/mount-isolation`
- `/fr/peak/mount-washington`

## Viewports
- `320x740`
- `768x1024`
- `1440x900`

## Core Checks
- Global nav is visible and interactive.
- Advisory banner spacing is clean above nav.
- Hero tools row shows print, share, unit switch, and directions controls.
- Buttons do not overlap, clip, or collapse at any breakpoint.
- Section rhythm is consistent across panels with clear dotted-row separators.

## Data Panel Checks
- `Standard Routes` shows all route rows.
- `Associated Trails` count matches source trail names.
- `Parking & Access` panel shows populated values and directions CTA.
- `Difficulty Metrics` panel renders technical and physical scores when available.
- `Risk & Preparation` panel renders risk factors/prep notes/weather snapshot when available.
- `Trail-tested notes` panel renders summary/tips/history when available.

## Media Checks
- Carousel controls work (`prev`, `next`, dot navigation, pause/play).
- Current slide image has non-empty `alt` and non-empty `title`.
- Metadata panel updates when carousel slide changes.
- No visible image stretching or clipping artifacts.

## Localization Checks
- FR route shows translated panel labels and CTA copy.
- Proper nouns (peak names/range names) remain accurate.
